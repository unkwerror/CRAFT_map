# Импорт объектов из monuments.json в БД карты

## Контекст

`monuments.json` — 100 памятных объектов Тюмени, извлечённых из рабочей доски КРАФТ. Структура записи:

```json
{
  "id": 1,
  "title": "Памятник морякам России",
  "category_id": "patriotism",          // patriotism | memory | dignity | continuity
  "category_title": "Патриотизм",
  "district": "Центральный округ",       // Центральный | Восточный | Калининский | Ленинский
  "address_raw": "Заречный парк",        // локация как записана на доске, может быть null
  "description": "…",                    // может быть null
  "lat": null, "lng": null,
  "geocode_status": "pending",
  "geocode_query": "Тюмень, Заречный парк",
  "flags": []
}
```

Координат в исходнике НЕТ — их нужно получить геокодингом и проверить вручную. Это ключевая часть задачи.

## Шаг 1. Импорт в БД

Записать все 100 объектов в таблицу `objects` (схема в CLAUDE.md):
- `title` → title, `description` → description, `category_id` → category_id
- `address_raw` → address
- `district` — НЕ доверять слепо: после геокодинга пересчитать через `ST_Contains(districts.geom, geom)` и сравнить с заявленным; расхождения — в отчёт
- `geom` — пока NULL, `published = false` (публиковать только после проверки координат)
- `flags` сохранить в отдельное jsonb-поле `import_flags` — они понадобятся при ручной проверке

## Шаг 2. Геокодинг

Скрипт `db/import/geocode.py`:

1. Для каждого объекта с `geocode_status = pending` выполнить запрос к **Nominatim** (`https://nominatim.openstreetmap.org/search?q={geocode_query}&format=json&limit=3&countrycodes=ru`). Соблюдать rate limit 1 req/sec и указывать User-Agent проекта.
2. Валидация результата: точка обязана попадать в bbox Тюмени (примерно 57.05–57.25 N, 65.35–65.75 E). Вне bbox → результат отбрасывается.
3. Классификация уверенности:
   - `high` — Nominatim вернул тип monument/memorial или точный адрес с домом
   - `medium` — найден только объект-контейнер (сквер, парк, площадь): координата центра контейнера допустима как первое приближение, ставим её
   - `failed` — ничего валидного
4. Записать lat/lng и `geocode_status` в БД. Сгенерировать отчёт `geocode_report.md`: таблица объект → статус → что нашлось.

Альтернатива/дополнение: пробовать вторым проходом геокодер 2ГИС или Яндекс.Геокодер (нужен API-ключ, у Яндекса есть бесплатная квота) для объектов со статусом failed — они заметно лучше знают русские адреса и POI.

Ожидаемо: ~19 объектов вообще без адреса (geocode_query построен из названия) и часть «мягких» локаций («район жд», «территория жд») дадут failed или medium — это нормально.

## Шаг 3. Ручная проверка

В админке сделать режим «Проверка импорта»:
- список объектов, отсортированный: failed → medium → high
- для каждого: мини-карта с текущей точкой (если есть), клик по карте = установить/поправить координату, кнопка «Подтверждено» → `geocode_status = verified`, `published = true`
- объекты с флагом `board_note_net_na_karte` — на доске помечены «НЕТ НА КАРТЕ» (заказчик не нашёл их на карте): требуют уточнения у КРАФТ
- флаг `possible_duplicate_of_same_monument_FD20-3031` — два дублирующих описания паровоза ФД у ДК «Железнодорожник»: показать оба, дать кнопку слияния/удаления
- флаг `from_board_sticky_note` — объекты добавлены со стикеров без описаний и адресов: собрать данные отдельно

## Шаг 4. Фото

Фото выгружены с доски: архив `monument_photos.zip` (папка `photos/`, 92 файла `obj_{id}_{n}.jpg`). В `monuments.json` у каждого объекта поле `photos` содержит имена его файлов; у 8 объектов (стикеры и карточки без фото) массив пуст.

При импорте:
1. Распаковать архив, для каждого фото прогнать через sharp: оригинал ≤1600px по длинной стороне + превью 400px, конвертация в webp, положить в `/data/uploads/objects/{id}/`.
2. Записать пути в jsonb-поле `photos` объекта в формате из CLAUDE.md (`[{original, thumb, alt}]`, alt = title объекта).
3. Учесть: фото собирались заказчиком с открытых источников, качество и разрешение неоднородные (40 КБ – 4 МБ). Объект с флагом `board_note_photo_placeholder` (ТЭЦ-2) — на доске отмечено, что фото временное. В админке фото должны свободно заменяться — КРАФТ сможет обновить любое.

## Известные особенности данных

- Тексты описаний местами содержат обрезанные последние предложения (карточки на доске не всегда дописаны) — это не ошибка парсинга, так в исходнике.
- Один и тот же объект может фигурировать в двух округах под разными названиями (например, «Памятник Б.Е. Щербине» стикером в Центральном и «Памятник Герою Соц. Труда Борису Щербине» в Восточном, сквер Немцова) — при проверке смотреть на пары с похожими именами; после геокодинга дубли всплывут по близости координат (`ST_DWithin`, 100 м).
- Категории (809-й Указ): patriotism = Патриотизм, memory = Историческая память, dignity = Достоинство, continuity = Преемственность поколений. Цвета — по легенде презентации КРАФТ.
