# Карта памятников — задачи на реализацию

> Спецификация по итогам планёрки 11.07.2026.
> Проект: интерактивная веб-карта памятников (аналог Яндекс.Карт), привязка к Тюмени, готовится под грант.
> Дедлайн первой итерации: **понедельник** (предварительная оценка реализуемости + часть функционала).

## Контекст проекта

- Уже реализовано: карта с памятниками (первичное наполнение сделано машинным путём), карточка памятника с одним фото, шаблон редактирования карточки для контент-менеджера (название, категория, адрес, описание, фото), кнопка «Маршрут в Яндекс.Картах» (переброс в приложение Яндекс.Карт с построением маршрута).
- Контентом (редактирование карточек) занимается отдельный человек — функционал редактора должен оставаться простым.
- Вход на ресурс — **без регистрации** (общедоступный ресурс). Собственной системы аккаунтов нет.

---

## 1. Поиск (приоритет: высокий)

Добавить строку поиска на карту.

**Требования:**
- Расположение: вверху по центру карты.
- Поиск должен находить памятники по трём типам запросов:
  1. **По округу/району** — например, «Калининский район» → показать памятники этого района на карте.
  2. **По теме/категории** — например, «исторические памятники» → отфильтровать карту по категории.
  3. **По названию памятника** — например, «Памятник Петру Первому» → найти и отметить конкретный памятник.
- Поведение как «умный поиск»: одна строка, тип запроса определяется автоматически (можно начать с простого варианта: поиск по индексу название + категория + район, с подсказками/автодополнением).
- Ориентир по UX — поиск в Яндекс.Картах.

**Связанные изменения UI:**
- Существующие фильтры «Памятные объекты» пересмотреть: частично убрать или переработать, чтобы не дублировали поиск.
- Блок «Статистика» (объекты по округам) с главного экрана **убрать**; если нужен — придумать, как показать органично (не в текущем виде).

## 2. Карточка памятника (приоритет: высокий)

Расширить карточку памятника.

**Требования:**
- **Медиа:** несколько фотографий (галерея) вместо одной. Поддержка фото уже есть «из коробки» — включить/довести. Добавить поддержку **видео** в ту же галерею (как в Яндекс.Картах: фото и видео вперемешку).
- **Рейтинг:** звёздочки, как в Яндексе. ⚠️ Источник рейтинга не решён (см. «Открытые вопросы») — на первом этапе достаточно UI-заглушки/поля в данных.
- **Описание:** текстовое описание. Заложить возможность разбивки на секции (например, «Архитектура», «История») — структура секций будет уточнена, делать расширяемо.
- **Аудиогид:** маленькая кнопка внутри карточки, проигрывающая аудио-сопровождение. Информация должна быть доступна в двух форматах: аудио + текст.
- **Информационный блок (правая панель карточки): увеличить размер** — сейчас слишком маленький, должен вмещать больше информации.

**Редактор карточки (для контент-менеджера):**
- В существующий шаблон редактирования добавить поля: загрузка нескольких фото, загрузка видео, загрузка аудио (аудиогид), секции описания.

## 3. Сбор средств (приоритет: средний)

- Добавить в карточку памятника функцию **«Пожертвовать»** — пожертвование на реставрацию памятника.
- Механизм оплаты/реквизиты уточнить у заказчика; на первом этапе — кнопка + страница/модалка с деталями.

## 4. Мероприятия у памятников (приоритет: средний)

Показывать на карте события, проходящие у памятников/в скверах (9 мая, День десантника, День народного единства и т.п.).

**Требования:**
- Данные вводятся и обновляются **вручную** администратором (список, аналогично памятникам). Автоматический сбор из внешних источников — **не делаем** (отложено: данные разрозненны, маппинг на памятники нереализуем сейчас).
- Модель данных: мероприятие привязано к памятнику/локации, имеет дату (или диапазон дат), название, описание.
- Отображение: при открытии карты метки памятников с мероприятием **на сегодняшнюю дату** визуально выделяются (мигание/пульсация), по клику подгружается информация о мероприятии.
- Админ-форма для ручного ввода/редактирования мероприятий.

## 5. Брендинг (приоритет: низкий)

- Добавить **герб Тюмени** в угол интерфейса карты — чтобы была видна привязка к городу.
- Крафтового оформления **не будет** — не закладывать.
- Файл герба и лекала предоставит заказчик (ждём от Юлии).

## Явно НЕ делаем в этой итерации

- ❌ 3D-модели памятников — отказались, работаем в текущем контексте.
- ❌ Собственные тур-маршруты по памятникам (тематические, по районам, ИИ-гид с геолокацией) — отложено; маршруты только через существующий переброс в Яндекс.Карты.
- ❌ Автоматический сбор мероприятий из внешних источников — только ручной ввод.
- ❌ Полноценная система регистрации пользователей — вход без регистрации.

## Открытые вопросы (заблокированы решением заказчика — не реализовывать без уточнения)

| Вопрос | Статус |
|---|---|
| Рейтинг: кто и как ставит (пользователи карты? откуда данные?) | Не решено |
| Отзывы: без регистрации нет авторства. Варианты: без отзывов вовсе / обезличенные отзывы от любого посетителя | Не решено — ключевая проблема |
| Фотографии: брать имеющиеся с Яндекса или предоставит заказчик | Не финализировано |
| Структура секций описания (архитектура/история/...) | Юлия уточнит |
| Какая ещё инфраструктура нужна на карте «чтобы было интересно и удобно» | На проработку |

## План первой итерации (к понедельнику)

1. Строка поиска (базовая версия: название + категория + район, автодополнение).
2. Убрать блок статистики, увеличить правый информационный блок.
3. Пересобрать фильтры.
4. Галерея: несколько фото + поддержка видео в карточке.
5. Кнопка аудиогида в карточке (плеер + поле аудио в редакторе).
6. Кнопка «Пожертвовать» (UI, без платёжной интеграции).
7. Модель данных и админ-форма для мероприятий + выделение меток с событиями на текущую дату.
8. Герб Тюмени в углу (после получения файла от заказчика).
9. Краткая записка по реализуемости остального — для заказчика.

## Ожидается от заказчика (входные данные)

- Фото герба Тюмени + лекала (Юлия).
- Информация по грантам — что должно быть описано (Юлия, завтра/понедельник).
- Решение по рейтингу и отзывам.
- Решение по источнику фотографий.
- Структура секций описания.
